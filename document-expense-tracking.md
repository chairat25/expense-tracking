# เอกสารอธิบายระบบบันทึกรายรับ-รายจ่าย (Expense Tracking System)

## 📌 ภาพรวมของระบบ (System Overview)

**Expense Tracking System** คือ เว็บบันทึกรายรับ-รายจ่ายรายวันในรูปแบบ **Mobile-First Responsive Web Application** ที่ถูกพัฒนาขึ้นมาเพื่อแทนที่การจดบันทึกบน Google Sheets / Excel โดยออกแบบมาสำหรับการใช้งานผ่านสมาร์ทโฟนในชีวิตประจำวัน ช่วยให้การบันทึกธุรกรรมเป็นเรื่องรวดเร็ว สะดวก และแม่นยำ ไม่ตกหล่น

ระบบคำนวณยอดยกมา รายรับ รายจ่าย และยอดเงินคงเหลือให้อัตโนมัติแบบ **Real-time** พร้อมทั้งมีระบบบริหารจัดการยอดยกมาข้ามเดือน และระบบแบ่งเงินเก็บสะสม (Savings Ledger) เพื่อช่วยวางแผนการเงินในระยะยาวได้อย่างมีประสิทธิภาพ

---

## 🚀 จุดเด่นและแนวคิดการออกแบบ (Key Concepts & Highlights)

1. **Mobile-First & Instant Quick-Add Design**:

   - ออกแบบ UI ให้เหมาะกับการแตะกรอกบนหน้าจอมือถือได้ด้วยมือเดียว
   - กรอกตัวเลข เลือกหมวดหมู่ แล้วกดบันทึกได้อย่างรวดเร็วในไม่กี่วินาที
   - เคลียร์ฟอร์มอัตโนมัติและโฟกัสที่ช่องจำนวนเงินทันที เพื่อให้กรอกรายการถัดไปได้ต่อเนื่อง
2. **Real-time Balance & Month Carousel**:

   - แถบสลับเดือน (Month Strip) บนสุดของทุกหน้า ช่วยให้รู้เสมอว่ากำลังจัดการข้อมูลของเดือนไหน
   - แสดง 3 ยอดสำคัญประจำเดือน (**ยอดยกมา · ใช้ไป · คงเหลือ**) อัปเดตสดทุกครั้งที่มีการบันทึก
3. **Savings & Carryover Split (ระบบแบ่งเงินเก็บและยกยอด)**:

   - เมื่อปิดยอดสิ้นเดือน สามารถเลือกแบ่งเงินคงเหลือส่วนหนึ่งเข้า **"กระปุกเงินเก็บสะสม" (Savings)**
   - ยอดเงินส่วนที่เหลือหลังหักเงินเก็บ จะถูกยกไปเป็น **"ยอดยกมา" (Opening Balance)** ของเดือนถัดให้อัตโนมัติ
4. **Timezone Accuracy (Asia/Bangkok)**:

   - ระบบล็อกเวลาการทำงานและสรุปผลตามเขตเวลาประเทศไทย (`Asia/Bangkok`) ป้องกันปัญหาบันทึกรายการช่วงค่ำแล้ววันที่กระโดดข้ามวัน
5. **PWA (Progressive Web App) Support**:

   - สามารถกด "Add to Home Screen" ติดตั้งลงหน้าจอโฮมบนมือถือ (iOS / Android) ใช้งานได้เหมือนแอปเนทีฟ
6. **Enterprise-Grade Security & Row Level Security (RLS)**:

   - ข้อมูลของผู้ใช้ทุกคนถูกแยกอย่างเด็ดขาดในระดับฐานข้อมูลด้วย Postgres Row Level Security (RLS) ต่อให้เข้าถึงผ่าน API Direct ก็ไม่สามารถอ่านหรือแก้ไขข้อมูลของผู้ใช้อื่นได้

## ✨ ฟีเจอร์หลักของระบบ (Main Features)

### 1. 🔐 ระบบยืนยันตัวตนและความปลอดภัย (Authentication & User Management)

- **Multi-Authentication Support**: รองรับการลงชื่อเข้าใช้งานด้วย **Email / Password** และ **Google OAuth**
- **Session Management**: จัดการ Cookie Session ผ่าน Supabase Auth ปลอดภัย ป้องการสวมรอย
- **Account Bar**: แสดงสถานะผู้ใช้งานปัจจุบัน และปุ่มออกจากระบบ (Logout)

---

### 2. 📅 แถบสลับเดือนและภาพรวมสรุป (Month Carousel / Month Strip)

- **Easy Navigation**: สลับเดือนได้ง่ายด้วยการปัดซ้าย-ขวาบนมือถือ (Swipe Gesture) หรือกดปุ่มลูกศร
- **Current Month Highlight**: ไฮไลต์เดือนปัจจุบันชัดเจน และป้องกันการกรอกข้อมูลในเดือนอนาคต
- **Real-time Monthly Summary**:
  - **ยอดยกมา (Opening Balance)**: เงินตั้งต้นที่มี ณ ต้นเดือน
  - **ใช้ไป (Total Expenses)**: ยอดรวมรายจ่ายทั้งหมดในเดือนนั้น
  - **คงเหลือ (Remaining Balance)**: เงินคงเหลือสุทธิ
- **Closed Month Indicator**: แสดงป้ายสัญลักษณ์ `✓ ปิดยอดแล้ว` สำหรับเดือนที่ทำการปิดบัญชีสิ้นเดือนเรียบร้อยแล้ว

---

### 3. ☀️ มุมมองรายวัน (Day View) — หน้าหลักสำหรับการใช้งานประจำวัน

- **Quick Add Form (ฟอร์มบันทึกด่วน)**:
  - ระบุจำนวนเงิน (Amount)
  - เลือกประเภท: **รายรับ (Income)** หรือ **รายจ่าย (Expense)**
  - เลือกหมวดหมู่ (Category):
    - 🍔 อาหาร (Food)
    - 🧋 เครื่องดื่ม (Drink)
    - 🚗 การเดินทาง (Transport)
    - 🧾 ค่าใช้จ่าย/บิล (Bill)
    - 🛍️ ช้อปปิ้ง (Shopping)
    - 🎬 ความบันเทิง (Fun)
    - 📦 อื่นๆ (Other)
  - เพิ่มหมายเหตุ (Note) เช่น ชื่อร้านค้า หรือรายละเอียดเพิ่มเติม
- **Daily Starting Balance Prompt**: ระบบแจ้งเตือนให้ระบุเงินตั้งต้น/รายรับแรกของวัน (หากยังไม่มีรายการรายรับในวันนั้น)
- **Transaction History Timeline**:
  - แสดงรายการธุรกรรมประจำวัน จัดเรียงตามเวลาจริง (`spentAt`)
  - แสดงเวลาที่บันทึกรายการ (เช่น 12:30 น.)
  - สามารถลบรายการธุรกรรมย้อนหลังได้พร้อมคำนวณยอดใหม่ทันที

---

### 4. 📊 มุมมองรายเดือน (Month View) — สรุปผลและวิเคราะห์การเงิน

- **Monthly Spreadsheet Summary**: ตารางสรุปภาพรวมทั้งเดือนในรูปแบบที่คุ้นเคย (`วันที่ | รายการ | รายรับ | รายจ่าย | หมายเหตุ | คงเหลือ`)
- **Category Expense Breakdown**: สรุปสัดส่วนการใช้จ่ายแยกตามหมวดหมู่ ช่วยให้เห็นว่าเสียเงินไปกับหมวดไหนมากที่สุด
- **Month Closing & Savings Split (การปิดยอดสิ้นเดือนและแบ่งเงินเก็บ)**:
  - ปุ่มกดปิดยอดสิ้นเดือน (Close Month)
  - กำหนดจำนวนเงินที่ต้องการแบ่งเก็บเข้ากระปุกเงินเก็บสะสม
  - ยกยอดเงินคงเหลือสุทธิหลังหักเงินเก็บไปเป็นค่ายอดยกมาของเดือนถัดไป

---

### 5. 💰 กระปุกเงินเก็บสะสม (Savings Ledger System)

- **Total Savings Tracker**: บันทึกและคำนวณยอดเงินเก็บสะสมรวมทั้งหมดของผู้ใช้
- **Savings Ledger History**: เก็บบันทึกประวัติการฝากเงินเข้าเงินเก็บจากการปิดยอดแต่ละเดือน

---

### 6. 📱 Progressive Web App (PWA)

- **Install Prompt Card**: การ์ดแนะนำการติดตั้งเลื่อนขึ้นมาให้อัตโนมัติเมื่อเข้าใช้งานผ่านเว็บบราวเซอร์บนมือถือ
- **Standalone App Experience**: เมื่อติดตั้งแล้วจะรันในโหมดแอปพลิเคชันเต็มหน้าจอ ไม่มีแถบ URL ของบราวเซอร์มารบกวน

## 🗄️ โครงสร้างข้อมูลหลัก (Database Schema Overview)

1. **`months`**: เก็บรายละเอียดภาพรวมของแต่ละเดือน
   - `userId`, `ym` (`YYYY-MM`), `openingBalance` (ยอดยกมา), `closedAt` (เวลาปิดยอด), `savingsAmount` (ยอดเงินที่แบ่งเก็บ)
2. **`transactions`**: เก็บรายการรายรับและรายจ่ายทั้งหมด
   - `userId`, `date` (`YYYY-MM-DD`), `spentAt` (Timestamp เวลาจริง), `type` (`income` | `expense`), `amount`, `category`, `note`
3. **`savings_transactions`**: บัญชีคุมยอดเงินเก็บสะสม (Savings Ledger)
   - `userId`, `amount`, `note`, `createdAt`
4. **`user_settings`**: ตั้งค่าการใช้งานของผู้ใช้
   - `userId`, `budgetMode` (`month` | `week`)
5. **`daily_budgets`**: เก็บค่าน้ำหนักงบประมาณรายวัน

## 🎯 สรุปคุณค่าของระบบ (Value Proposition)

ระบบบันทึกรายรับ-รายจ่ายนี้ถูกออกแบบมาอย่างประณีตเพื่อให้ **"การจดบันทึกการเงินประจำวันกลายเป็นเรื่องง่ายและทำได้จริงทุกวัน"** แก้ปัญหาเดิมๆ ของการใช้ Excel/Google Sheet บนมือถือ เช่น การพิมพ์ยาก หน้าจอเล็กลง ฟอร์มซับซ้อน หรือสูตรหลุด โดยมอบประสบการณ์การใช้งานที่ราบรื่น ปลอดภัย ปรับแต่งได้ตามความต้องการ และพร้อมขยายผลในอนาคต
